# Auto Key Presser

A compact Windows desktop application that repeatedly presses a selected keyboard key in the currently active window.

## Features

- Select a key or a `Shift` / `Ctrl` / `Alt` combination by pressing it physically.
- Use millisecond or second intervals (minimum 50 ms).
- Add optional random timing deviation.
- Configure a delayed start.
- Run without a limit, stop after a number of presses, or stop after a set time.
- Stop automatically when another physical key is pressed.
- See the press count and a live countdown to the next press.
- Dark, compact, English-only interface.
- Save settings in `%LOCALAPPDATA%\AutoKeyPresser\settings.json`.

The application uses Windows `SendInput` with a custom `dwExtraInfo` marker and a `WH_KEYBOARD_LL` hook. Events generated by the application are distinguished from physical keyboard events.

## Download

The self-contained Windows x64 executable is available at `dist/AutoKeyPresser.exe`. It does not require .NET to be installed.

## Requirements for development

- Windows 10 or 11
- .NET 8 SDK or Visual Studio 2022 with .NET desktop development support

## Run from source

```powershell
dotnet restore --configfile NuGet.Config
dotnet run --project AutoKeyPresser
```

## Tests

```powershell
dotnet test AutoKeyPresser.slnx --configfile NuGet.Config
```

## Publish as one EXE

```powershell
dotnet publish AutoKeyPresser/AutoKeyPresser.csproj -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true /p:IncludeNativeLibrariesForSelfExtract=true /p:DebugType=None /p:DebugSymbols=false -o dist --configfile NuGet.Config
```

## Limitations

The key is sent to the currently active window. The application does not select a target process or bypass application or game protections. A non-elevated instance may not send input to an elevated window, and some games or anti-cheat systems may ignore `SendInput`.
